Backup & Restore

⚠️ Why this matters on Render free tier

Render's free tier wipes the filesystem (and your SQLite database) on every redeploy, restart, or spin-down after inactivity. That means your plans, prices, services, payment methods, categories, promo codes, and shop/APK settings get reset to whatever ships in the code unless you back them up first and restore them after.

Not included in this backup (these live in the database and can't survive a Render free-tier reset no matter what — only a paid persistent disk or external DB fixes this): user accounts, balances, orders, deposits, and the keys pool.

📤 Backup setup

Saves everything below as one JSON file/snippet. Do this right before you redeploy or update the code.
  • Plans (prices, days, categories)
  • Services & tiers
  • Service categories
  • Payment methods (incl. QR code images)
  • Promo codes
  • Deposit settings (GCash number / QR)
  • Shop section toggles
  • APK app settings

📥 Restore setup

Upload a backup file you saved earlier. Existing items with the same ID/key/code are updated in place; new ones are created. This does not delete anything not present in the backup.
